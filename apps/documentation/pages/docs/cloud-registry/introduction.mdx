# Cloud Registry

The Elixir Cloud Registry package provides web components for managing and creating service registrations in the Elixir Cloud ecosystem. These components enable developers to register new services and manage existing service entries through an intuitive interface.

<Callout type='info'>
  This package is designed to work with Elixir Cloud Registry implementations and provides extended functionality beyond the standard GA4GH Service Registry specification.
</Callout>

## Installation

<Callout type='info'>
  The following examples use `cloud-registry` as the package name. You can find installation instructions in the [Installation Guide](/docs/installation).
</Callout>

## Package Information

- **Package**: `@elixir-cloud/cloud-registry`
- **Version**: `2.0.0-alpha.6`
- **Repository**: [GitHub](https://github.com/elixir-cloud-aai/cloud-components)

## Components

The Cloud Registry package includes the following components:

### Service Create Component
A comprehensive form component for creating new service registrations in the Cloud Registry, with validation and error handling.

## Key Features

- **Service Registration**: Complete form interface for registering new services
- **Validation**: Built-in form validation with real-time feedback
- **Advanced Fields**: Support for optional and advanced service metadata
- **Event-Driven**: Emits events for successful creation, failures, and validation errors
- **Customizable**: Support for custom data providers and base URLs
- **Responsive Design**: Optimized for both desktop and mobile usage

## Use Cases

- **Service Onboarding**: Help service providers register their APIs
- **Administrative Tools**: Build management interfaces for service registries
- **Development Workflows**: Integrate service registration into CI/CD pipelines
- **Self-Service Portals**: Enable users to register their own services
