# TRS-Filer

The `TRS-Filer` package extends the base `TRS` components with write capabilities, allowing you to create new tool registrations. This package extends the GA4GH Tool Registry Service API to allow creation of resources.

## Package Information

- **Package**: `@elixir-cloud/trs-filer`
- **Version**: `2.0.0-alpha.6`
- **Standard**: [Elixir Cloud & AAI v2.1.0](https://github.com/ga4gh/tool-registry-service-schemas/blob/release-2.0.1/openapi/openapi.yaml)

## Features

- **Tool Creation**: Provides a form to register new tools by submitting metadata.
- **Validation**: Includes client-side validation for required fields.
- **Extensible**: Can be used with any TRS API that supports `POST` requests for tool creation.

## Available Components

- [`<ecc-client-elixir-trs-filer-tool-create>`](./components/tool-create): A component for registering new tools.
- [`<ecc-client-ga4gh-trs-tools>`](../trs/components/tools): A component for listing existing tools.
- [`<ecc-client-ga4gh-trs-tool>`](../trs/components/tool): A component for displaying details of a specific tool.

*Note: The tools and tool components are from the [`trs`](../trs/introduction) package.*
