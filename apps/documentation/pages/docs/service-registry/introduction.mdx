# Service Registry

The GA4GH Service Registry package provides web components for interacting with GA4GH-compliant service registry implementations. These components enable you to browse, search, and view detailed information about registered services in a standardized way.

<Callout type='info'>
  This package implements the GA4GH Service Registry specification, allowing you to connect to any compliant service registry endpoint.
</Callout>

## Installation

<Callout type='info'>
  The following examples use `service-registry` as the package name. You can find installation instructions in the [Installation Guide](/docs/installation).
</Callout>

## Package Information

- **Package**: `@elixir-cloud/service-registry`
- **Version**: `2.0.0-alpha.6`
- **Repository**: [GitHub](https://github.com/elixir-cloud-aai/cloud-components)

## Components

The Service Registry package includes the following components:

### Services Component
Display a list of services from a GA4GH Service Registry with search and filtering capabilities.

### Service Component  
Show detailed information about a specific service, including metadata, organization details, and API specifications.

## Key Features

- **GA4GH Compliance**: Full compatibility with GA4GH Service Registry specification
- **Search & Filter**: Built-in search functionality and service type filtering
- **Responsive Design**: Works across desktop and mobile devices
- **Event-Driven**: Emits events for service selection and data changes
- **Customizable**: Support for custom data providers and styling
