# How to write Documentation

<Callout emoji='ðŸ¤£' type='info'>
  Welcome to documentation about documentation!
</Callout>

We welcome all kinds of contribution, and documentation is no exception. Cloud components's documentation (this very documentation), is written
using the [nextra](https://nextra.site/) framework, which is a wrapper around [Next.js](https://nextjs.org/). This means that you can write
documentation using Markdown, and also use React components to make your documentation more interactive and engaging.

The documentation is organized into sections, and each section is a folder in the `/apps/documentation/pages` directory. We aim to write documentation,
so that its optimized for [SEO](https://nextjs.org/learn-pages-router/seo/introduction-to-seo), and also easy to navigate. Please follow the guidelines
of nextjs and nextra when writing documentation. But here is the overview of how to write documentation. 

All the non interactive component should be written in `mdx` format and define your component in the `/apps/documentation/components` folder. `pages` directory will
contain `mdx`, `_meta.json` is used to configure, rename, and add metadata to the page. Nesting or separation of concerns is done by creating folders in the `pages` directory.
